## 1. Block Overview
The AXI_DMA is a single-channel Direct Memory Access controller designed for memory-to-memory data transfers. It exposes an AXI4 master interface for performing source and destination memory accesses, and an APB slave interface for software configuration and status readback. The block supports configurable transfer sizes (1–65535 bytes), INCR-type burst transfers of up to 16 beats, and generates interrupts on transfer completion or AXI error. Scatter-gather is explicitly not supported in this version (v0.9 Draft). The block is positioned as a simple, single-outstanding-transaction DMA engine suitable for use in SoC subsystems requiring lightweight memory-copy acceleration.

## 11. Known Constraints
- Only memory-to-memory transfers are supported; peripheral-to-memory and memory-to-peripheral are not supported
- Only a single outstanding AXI transaction is permitted at a time; no transaction pipelining
- Source and destination addresses must be word-aligned (32-bit / 4-byte aligned)
- Scatter-gather mode is not supported in this version
- Maximum transfer size is 65535 bytes
- Maximum burst length is 16 beats (INCR type only)
- Error state requires explicit software clear before a new transfer can be initiated
